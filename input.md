## Step6 input contracts

### Branch A: Reporter_Experiment

Source:

- pooled RNA, DNA, and pool metadata matrices

Input files:

- `pooling_scheme_3_RNA.csv`
- `pooling_scheme_3_DNA.csv`
- `pooling_scheme_3_meta.csv`

Output logic:

- reshape wide pool-level RNA/DNA columns into long format
- calculate CPM-normalized DNA and RNA within each replicate
- calculate `log2FoldChange = log2(rna_normalized / dna_normalized)`
- keep one row per `replicate x oligo_name`

Important current limitation:

- `n_bc` source is not fixed yet for this project version
- for now, `n_bc` will be left blank

### Branch B: Reporter_Element, Reporter_Variant, genomic BED outputs

Sources:

- pooled RNA matrix
- pooled DNA matrix
- MPRA element statistics
- MPRA variant statistics

Output logic:

- `Reporter_Element.tsv.gz`
	- one row per tested oligo
	- uses `pooling_scheme_3_mpra_element.csv`
- `Reporter_Variant.tsv.gz`
	- one row per tested variant
	- uses `pooling_scheme_3_mpra_variant.csv`
- genomic BED outputs
	- derived from element or variant results
	- genomic fields are currently placeholders until the final annotation source is fixed

## Fields intentionally left blank for now

The following unresolved fields are intentionally emitted as blank values in the
first GitHub-ready formatter version:

1. `Reporter_Experiment.n_bc`
2. genomic coordinate fields that require a finalized annotation source
3. `Reporter_Variant.postProbEffect`, `CI_lower_95`, `CI_upper_95` when not available from the current project result set
